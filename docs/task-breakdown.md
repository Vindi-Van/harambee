# Task Breakdown (v1)

## Track A — Project Operating System

1. Define GitHub labels and stage taxonomy
2. Create issue templates (feature, bug, QA return, blocker)
3. Create discussion templates (dispatch, standup, escalations)
4. Define assignment protocol (worker request -> Oga assignment)
5. Define transition policy (required artifacts per stage)

## Track B — Role Contracts

1. Finalize OgaArchitect contract
2. Finalize UI agent contract (design/brand/accessibility requirements)
3. Finalize Coding agent contract (tests/build/quality requirements)
4. Finalize QA agent contract (verification matrix + reject criteria)
5. Finalize DevOps contract (release and rollback requirements)

## Track C — Quality & Complexity

1. Publish complexity rubric examples (1–5)
2. Publish decomposition policy for C4/C5
3. Define reserved fix window policy and defaults
4. Define anti-context-corruption checklist

## Track D — Pilot Execution

1. Pick pilot feature (example: login)
2. Run full lifecycle with artifacts at each stage
3. Force one QA rejection to validate bounce-back path
4. Capture lessons learned
5. Update contracts/rules

## Track E — Optional Coordination Service

1. Add Redis coordination spec (leases/heartbeats)
2. Define Oga ownership of coordination keys
3. Define reclaim policy for stale tasks
4. Add failure-mode test plan
